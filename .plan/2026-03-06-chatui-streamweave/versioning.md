# Asset Versioning and Dev Mode

## Goal

- Zero-config single-binary deployment: assets extracted to disk on first run
- Auto-update: when a new clawq version starts, UI assets on disk are refreshed
- Dev mode: bypass auto-overwrite so UI can be edited without OCaml rebuild

## Version Hash

`Chat_ui_assets.ui_version` is a SHA-256 content hash computed over the bundled JS + CSS,
generated by `scripts/gen_chat_ui_assets.sh` and embedded as an OCaml string constant.

Format: `"sha256:<hex64>"` — a deterministic, reproducible identifier.

## Server Startup Logic (`Ui_server.init`)

```
ui_dir  = $HOME/.clawq/ui/
dev_file = ui_dir + "DEV"
ver_file = ui_dir + "VERSION"

1. If dev_file exists → skip all checks; serve from ui_dir
2. Read ver_file → stored_version (empty string if missing)
3. If stored_version = Chat_ui_assets.ui_version → version matches; serve from ui_dir
4. Else → extract all assets to ui_dir:
     write index.html, chat.js, chat.css from Chat_ui_assets.*
     write ver_file with Chat_ui_assets.ui_version
   → serve from ui_dir (now up to date)
```

This runs synchronously at daemon/gateway startup. Extraction is O(KB), negligible cost.

## Asset Serving (`Ui_server.serve_asset`)

```
path: string  (e.g. "chat.js")
1. Try read ui_dir/path → return file contents + mime type
2. On ENOENT → fall back to embedded Chat_ui_assets string
3. On other error → log warning; fall back to embedded
```

## Dev Mode

Enable by creating `~/.clawq/ui/DEV` (empty file):

```sh
mkdir -p ~/.clawq/ui
touch ~/.clawq/ui/DEV
```

Then copy built assets there:
```sh
cp ui/dist/chat.js ui/dist/chat.css ui/dist/index.html ~/.clawq/ui/
```

Or use `make ui-dev` which watches for changes and auto-copies.

With DEV mode active, clawq serves whatever files are in `~/.clawq/ui/` without overwriting them.
No OCaml rebuild needed during UI development.

To exit dev mode: `rm ~/.clawq/ui/DEV` → next startup restores embedded assets.

## Client-Side Version Check

The served `index.html` includes:
```html
<meta name="ui-version" content="sha256:abcdef...">
```

`version.ts` logic:
1. On page load: read meta content → `localVersion`
2. On `visibilitychange` (tab refocused) and every 5 minutes: `GET /ui-version`
3. If response.version != localVersion → show update banner
4. Banner: "clawq updated — [Reload]  [Dismiss]"
5. Reload button: `location.reload()`
6. Dismissed flag stored in `sessionStorage` (cleared on new tab/reload)

This handles the case where clawq daemon restarts with a new version while the tab is open.

## Version in CI

`make ui-check` runs `gen_chat_ui_assets.sh --check`:
- Rebuilds UI assets into a temp dir
- Compares to committed `chat_ui_assets.ml`
- Exits non-zero if they differ (committed assets out of sync with source)

This is optional — the embedded assets are only stale if someone edited `ui/src/` without
running `make ui`. CI can enforce this if desired.
